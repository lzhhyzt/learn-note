一，创建版本库 1. 注意 1. 包括git在内的所有版本控制系统，都只能追踪文本文件的改动。二进制文件可以版本控制但是不能知道具体改了啥 2. 文件夹里面有个 .git 文件夹就是版本库 3. git每个命令后面带上文件名表示指定文件(文件名可以是 . 或 *)，否则就是全部文件 2. 创建版本库 1. 新建一个文件夹 mkdir learngit 2. 进入文件夹 cd learngit 3. 初始化git仓库 (如果文件夹不是空的也可以) git init 3. 把文件添加到版本库 1. 编写文件 readme.txt ，添加一些内容 git is a version control system git is a free software 2. 把工作区推送到暂存区 (文件可以有多个，用空格隔开) 1. 用法 1. 添加所有改动 git add -A 2. 添加新建文件和修改，但是不包括删除 git add * 或 git add . 3. 添加修改和删除，但是不包括新建文件 git add -u 4. 删除暂存区文件 git rm filename 2. 例子写法 git add readme.txt 3. 把暂存区推送到仓库，-m 参数是提交说明 git commit -m "commit a readname.txt" 4. 查看当前状态 1. 修改下readme.txt的内容，在下面命令用完后提交 git is a distributed version control system git is a free software 2. 查看工作区和暂存区对比的状态 git status 3. 查看工作区和暂存区对比的修改情况 git diff readme.txt 4. 查看工作区和指定版本库对比的修改情况 git diff HEAD -- readme.txt 5. 工作区和暂存区 1. git add命令将工作区提交到暂存区，git commit才真正将暂存区放到版本库 2. 暂存区颜色 红色 工作区和暂存区对比，不是暂存区的一部分 绿色 暂存区和版本库的对比，真正暂存区的一部分 没颜色 工作区和暂存区一样
