一，自定义git 1. 配置项 1. 例如，让git显示颜色 git config --global color.ui true 2. 配置别名(使用alias) 1. 例如配置 git last 为 git log -1 git config --global alias.last 'log -1' 2. 配置文件 (可手工更改 [alias] 配置) 1. 每个仓库的配置文件 1. vim .git/config 2. 后面增加内容 [alias] last = log -1 2. 当前用户的git配置文件 1. 在用户主目录的.gitconfig文件中 vim ~/.gitconfig 2. 忽略特殊文件 1. 例子 1. 工作区创建两个文件 touch a.txt b.txt 2. 工作区根目录创建文件 .gitignore touch .gitignore 2. 按照每行写上需要忽略的文件/文件夹，支持*通配符，写完马上就会生效 vim .gitignore 写上内容： *.txt 3. 把文件 .gitignore文件也提交到版本库，可以做版本控制 4. 添加一个 .gitignore忽略的文件报错 1. 强制添加， add加上-f参数 git add -f a.txt 2. 使用 check-ignore命令检查某个文件的忽略情况 (查看.gitignore第几行限制了) git check-ignore -v a.txt
