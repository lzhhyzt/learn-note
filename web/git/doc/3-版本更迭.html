一：时光机穿梭 1. 版本回退 1. 再次修改文件并提交 git is a distributed version control system git is a free software distributed under the GPL 2. 查看commit记录，按照最近(当前)到最远的方式 1. 显示每次的分支节点信息 ( commit ID(快照)，时间以及commit提交的-m说明 ) git log 2. 简洁显示分支节点信息 ( commit ID 和-m说明 )，加上--pretty=oneline参数 git log --pretty=oneline 3. 版本回退 1. git reset [--hard] 版本 [ filename ] 1. 解释 1. git内部有个指向当前版本的HEAD指针，回退版本只是将HEAD指针指向要想的快照 2. 用版本库替换暂存区，--hard参数表示同时也替换工作区 3. filename不写表示全部文件都回退 2. 版本 解释 1. HEAD表示当前版本 2. HEAD^表示上一个版本，上上版本就是HEAD^^，往上100个版本HEAD~100 3. commit ID的值，只需前7,8位能找到就行了 2. 查看版本切换的记录，由近到远 (可以用来查找 commit ID) git reflog 3. 例子写法 1. 回退到上一个版本 git reset --hard HEAD^ 2. 查看当前工作区的状态(commit 记录) git log 3. 回退到版本库之前的版本 () 1. 查看版本切换的记录找到之前的commit ID git reflog 2. 根据找到的commit ID 回滚到之前的那个版本 git reset --hard e547a08f7c 3. 撤销指定文件修改 git checkout -- [文件名/.] 1. 用暂存区替换工作区(绿色的才是暂存区)
